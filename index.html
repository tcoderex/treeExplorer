📖 Windows Family Tree Interactive User Tutorial

Comprehensive step-by-step masterclass on adding and auto-linking members in the engine.

The manual form is optimized for absolute control. It features relationship auto-linking, spouse pairings, maternal links, and self-healing cycle prevention:

  • Given Name & ID: Specify the individual's name and an optional unique bracketed custom ID. If no ID is provided, the engine auto-generates one.
  • Father's & Mother's Name: Specify their parents. The engine matches existing nodes in your tree. If they do not exist, it builds placeholders. Spouses are tracked to allow interactive child filtering in the visual tree.
  • Grandfather's Name: Links the father to the grandfather automatically to preserve lineage structure across generations.
  • Self-Healing Loop Prevention: The database engine analyzes relationships recursively on save. If any addition would create a loop (e.g. child becoming their own ancestor), it is blocked, and names are split into unique identifiers.

The bulk importer uses a robust, off-line regex parsing system designed for rapid database injection. It supports English, Arabic, and Semitic lineages, and CSV mappings:

1. English Lineage Strings (Recursive)

John son of Robert son of William [mother Jane]
Alice daughter of Robert son of William

The parser detects son of (Male) or daughter of (Female) delimiters, extracts mother names, and recursively resolves the tree path William ➔ Robert ➔ John/Alice.

2. Arabic / Semitic Lineage Chains

Ahmad bin Ali bin Hassan
Fatima bint Ali bin Hassan

Detects bin (Male) or bint (Female) delimiters and establishes the exact lineage connections seamlessly.

3. Comma-Separated CSV Formatting with IDs

ID, GivenName, FatherName, GrandfatherName, Gender(M/F), SpouseName

Examples:
• 55df, John, Ahmad (Alphanumeric ID)
• 1, John, Ahmad (Numeric ID)
• aaaaa-aaa-a1, John, Ahmad (Hyphenated ID)
The engine automatically identifies the first column as the unique ID if it contains at least one digit.

4. Declaring Unique IDs in Linear Chains (Parentheses & Brackets)

(jm-1) John son of (rm-1) Robert son of (wm-1) William
[sm-1] Sarah daughter of (rm-1) Robert son of (wm-1) William

You can prefix any name in a lineage chain with a parenthesized (id) or bracketed [id] unique identifier containing at least one digit. This maps connections cleanly in memory and prevents duplicate-name collisions!
